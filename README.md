# Lexical Scanner and Tokenizer

A C++ project demonstrating the **Lexical Analysis** phase of a compiler.  
The project reads raw C++ source code, converts it into meaningful symbols called *tokens*, and identifies any illegal characters.

---

## Overview
This project focuses on the **Lexical Analysis** phase of a compiler. Its primary goals are:

- Read raw source code
- Break it into a stream of meaningful symbols (*tokens*)
- Identify illegal or unexpected characters

The scanner ensures proper handling of whitespace, comments, and different token types.

---

## Core Working Logic
The scanner uses a **two-pointer buffering technique** (`lexemeBegin` and `forward`) to efficiently navigate through the input text.

### Token Recognition
It categorizes text into different token types:

- **KEYWORD** (e.g., `int`, `if`, `else`)  
- **IDENTIFIER** (variable/function names)  
- **OPERATOR** (e.g., `+`, `=`, `<<`)  
- **SEPARATOR** (e.g., `;`, `(`, `)`)  
- **INT_CONST** (integers like `10`)  
- **FLOAT_CONST** (decimals like `2.5`)  

### Handling Whitespace & Comments
- Ignores spaces, tabs, and multi-line comments  
- Supports string literals enclosed in double quotes  

### Error Detection
If the scanner encounters a character that **does not match any category** (e.g., `?`), it logs a **Lexical Error** with the specific line number.
## Files in the Project
- **scanner.cpp** → Main lexical scanner using two-pointer technique  
- **file2.cpp** → Alternative tokenizer using Regular Expressions  
- **input.txt** → Sample C++ source code to test the scanner  
- **tokens.txt** → Output generated by the scanner
